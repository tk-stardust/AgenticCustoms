# 《基于Agentic RAG与多智能体协作的跨境合规贸易与关税智能申报平台》

## 一、这个项目是做什么的

**通俗一句话定义**：

这是一个**AI外贸合规助手**，通过**Agentic RAG**精准检索各国海关税则、HS编码注释、制裁清单，再通过**多个智能体分工协作**，帮助中小外贸企业自动完成**商品归类、关税计算、禁限品校验、申报文件生成**，降低合规风险与报关成本。

**区别于普通AI问答平台的核心亮点**

1. 普通RAG：用户问啥、检索啥、直接回答，**不会思考、不会拆解任务、不会自主干活**；
2. **Agentic RAG**：具备智能规划能力，能**自主分析商品描述、判断需要查哪些税则章节、多轮检索、纠错优化、溯源税则条文**，归类更专业、可解释；
3. **多智能体协作**：拆分不同AI角色，各司其职，模拟关务团队分工，**一个AI负责HS编码推理、一个负责关税计算、一个负责合规校验、一个负责原产地匹配、一个负责生成申报文件**，比单一大模型更专业、模块化更强。

## 二、项目背景与痛点

### 2.1 行业现状痛点

- **HS编码错一个数字，退运+罚款**：中小外贸企业人工归类错误率高，一个数字差异可能导致关税相差数倍甚至被海关扣货；
- **多国法规碎片化**：东盟、欧盟、USMCA等关税规则、原产地标准、制裁名单实时变化，企业难以跟进；
- **传统软件只查表，不推理**：不能根据产品描述动态判断，需要用户先知道编码才能查税率；
- **合规与成本矛盾**：想利用FTA优惠税率，但不会匹配原产地规则；
- **申报文件繁琐**：每票货物需要准备报关单、原产地证书、合规声明，重复劳动多、易出错。

### 2.2 项目解决的核心问题

用 **Agentic RAG + 多智能体协作** 解决：**外贸企业“归类难、税率算不准、违规风险高、申报文件烦”** 的长期痛点。

## 三、系统整体架构

整体四层架构：**数据层 → Agentic RAG智能层 → 多智能体业务层 → 前端展示层**

### 1、数据层（知识库）

搭建跨境贸易知识库，通过公开数据源获取：

- 各国海关税则（HS编码、商品描述、基础税率、增值税、反倾销税）；
- WCO HS注释（归类总规则、类章注释）；
- 原产地规则文本（FTA协定中的原产地标准）；
- 制裁实体清单、禁限品目录；
- 报关单模板、原产地证书模板。

处理动作：切片、向量化、入库（向量数据库），同时将结构化数据存储至MySQL。

### 2、核心智能层：Agentic RAG 引擎

整个系统的大脑，负责所有智能推理：

- 用户意图识别（区分归类查询/关税试算/合规校验/申报文件生成）；
- 复杂问题拆解（如“这款带蓝牙功能的智能手表应该归哪一类？”→拆解出材质、功能、用途子问题）；
- 动态多轮检索（先查类章注释，再查具体品目，必要时查历史归类案例）；
- 答案合规校验、幻觉抑制、条文溯源。

### 3、业务层：五大外贸智能体

模拟真实关务团队分工，设计**5个独立智能体**，协同完成完整业务流程：

#### 智能体1：HS编码推理智能体

根据商品名称、材质、功能、用途等描述，结合WCO注释与历史归类案例，推理出最可能的HS编码（前6位国际通用），并给出推理路径和置信度。支持不确定时主动追问用户关键属性。

#### 智能体2：关税与税费计算智能体

基于目标国家税则，自动计算：基础关税 + 增值税 + 反倾销税 + 优惠税率（FTA），支持不同原产地场景比价，输出税费明细表。

#### 智能体3：合规与禁限品校验智能体

检索目标国家的禁止进口清单、许可证要求、环保合规（如RoHS、REACH）、制裁名单，标识“红/黄/绿”风险等级，并给出违规后果说明。

#### 智能体4：原产地规则匹配智能体

根据商品HS编码、生产工序、成分比例，判断是否符合FTA原产地标准（如区域价值成分RVC、税则归类改变CTC），推荐最优原产地申报策略。

#### 智能体5：申报文件生成智能体

自动生成：报关单草单、原产地证书申请书、合规声明、物流单证对照表，并做交叉校验，确保各文件数据一致。

### 4、前端展示层

用户Web端界面：商品输入（文字描述或上传图片OCR辅助）、一键试算关税、风险看板、申报文件下载、历史记录查询。

## 四、核心业务流程

**完整闭环流程（6步）**：

1. **用户输入**：用户输入商品描述（如“智能蓝牙音箱，塑料外壳，内置Wi-Fi，用于家庭音乐播放”）或上传产品图片；
2. **智能路由**：系统识别意图，路由至**HS编码推理智能体**；
3. **Agentic RAG推理**：拆解商品特征 → 多轮检索WCO注释、历史归类案例 → 输出HS编码及置信度；
4. **并行处理**：自动触发三个智能体并行工作
   - **关税计算智能体**：计算基础关税+附加税+增值税+优惠税率
   - **合规校验智能体**：检查禁限品、许可证、制裁名单
   - **原产地匹配智能体**：判断是否符合FTA优惠条件（如需）；
5. **文件生成**：**申报文件生成智能体**汇总所有结果，输出报关单草单、原产地证书申请书、合规声明；
6. **前端展示**：展示编码结果、税费明细、风险等级、可下载文件，并保存至历史记录。

## 五、项目创新点

1. **技术创新：Agentic RAG 赋能外贸归类**
   传统归类工具只能关键字匹配，本项目实现商品特征自主拆解、多轮检索、自我校验，所有结论可溯源至WCO注释或各国税则，显著降低归类错误风险。
2. **架构创新：多智能体分工协作**
   摒弃单模型一体化处理，采用专业化智能体分工（编码、计税、合规、原产地、申报），业务更精细、准确率更高，每个智能体可独立优化和扩展。
3. **业务创新：全场景外贸合规闭环**
   覆盖HS编码推理、关税试算、风险校验、原产地匹配、文件生成五大核心场景，真正实现从商品描述到申报文件的一条龙辅助。
4. **效果创新：抑制大模型幻觉**
   所有回答基于官方法条与真实案例库，强制溯源校验，解决AI在外贸合规领域“胡说八道”的致命问题，保障企业合规申报安全。

## 七、常见疑问

### 1、这个项目难度大吗？

**难度适中**：HS编码数据高度结构化，RAG检索可直接用向量相似度实现；多智能体之间多为顺序或并行调用，无复杂状态同步。基础功能易于实现，Agentic RAG与多智能体协作作为创新点有成熟框架支持，可快速搭建原型并逐步迭代。

### 2、和普通关税查询网站有什么区别？

普通网站只能输编码查税率；本系统**能根据商品描述自动推理编码、并行计算关税和合规校验、输出申报文件**——这是AI外贸合规的进阶形态，相当于一个7x24小时的初级报关员。

### 3、数据从哪来？

- HS编码+描述：中国海关总署官网公布的《进出口税则》；
- 税率：商务部外贸实务查询服务、各国海关官网；
- 制裁清单：美国OFAC、中国商务部不可靠实体清单（公开文本）。
  原型阶段使用预置常用商品HS编码及模拟税率，保证演示效果；生产环境可接入实时数据源。

### 4、支持哪些国家/贸易流向？

初期聚焦**中美、中欧、中国-东盟**三大主流贸易路线，覆盖约50个主要贸易伙伴。数据层设计为可扩展架构，后续可逐步增加更多国家和地区。

### 5、是否可以直接对接国际贸易单一窗口？

本项目后端预留标准化API接口，输出的报关单草单、原产地证书等文件格式符合中国国际贸易单一窗口的XML/JSON规范，后续可开发插件实现一键推送申报。

## 八、项目最终成果

- 一套基于Agentic RAG的HS智能推理引擎
- 五个专业化跨境贸易智能体协作系统（编码、计税、合规、原产地、申报）
- 多国法规知识库与向量检索库
- Web可视化贸易合规工作台（商品归类、关税试算、风险看板、文件生成）
- 完整项目文档、测试报告、创新说明
